1. Customer checkout

  1. Customer (role: customer) uses Post job.
  2. postJobAction creates a Stripe Checkout Session (app/dashboard/customer/post-job/actions.ts).
  3. Webhook payment_intent.succeeded or checkout.session.completed inserts jobs row (idempotent on PI id).
  4. app/api/confirm-checkout/route.ts — optional client confirmation path after return from Stripe.

2. Line Holder — Stripe Connect

  1. startStripeConnectOnboardingAction (app/dashboard/waiter/connect/actions.ts) creates Express account if needed and an Account Link.
  2. Account Link type: account_onboarding until both details_submitted and payouts_enabled are true in DB; then account_update for bank changes.
  3. User must complete Stripe’s flow including final Confirm on Review.
  4. Sync: webhook account.updated + manual Refresh Stripe status via refreshStripeConnectStatusAction / syncStripeConnectFromStripeForUser.

3. Payout to Line Holder

  1. finalizeJobPayout (lib/stripe-release-payout.ts): stripe.transfers.create for 80% of offered_price (20% platform fee in code).
  2. Stores jobs.payout_transfer_id; idempotent.

5. Sequence (checkout → job → transfer)

Customer → LineCrew: Post job + checkout LineCrew → Stripe: Checkout Session Customer → Stripe: Pay Stripe → Webhook: payment_intent.succeeded Webhook → DB: insert jobs Line Holder: Connect + accept job LineCrew → Stripe: transfers.create (when finalizeJobPayout runs) DB: job.payout_transfer_id set
